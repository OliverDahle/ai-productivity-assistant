# Personal Productivity Assistant
#### Video Demo: https://youtu.be/Jg7Gm4pTUJg
#### Description:

The **Personal Productivity Assistant** is a web application designed to help users organize and manage their time by generating daily productivity plans. Often I find myself struggling with time management, and this tool will help me with that. Instead of having to use valuable time planning my day to day, I can now simply input time available and the tasks at hand, and in two seconds a well formulated plan with my preferences is created.

The web application is built with **Flask** for the backend, with **Flask-Login** for user authentication, **Flask-SQLAlchemy** for database management, and **Flask-WTF** for form handling and security. It also uses **Bootstrap** for styling the front-end and making the application responsive. More on this later though.

## Why this project?
In general I wanted a project that included an API, some form of AI, and the Flask framework. When I asked myself how I could integrate all these components in to one project, I knew instantly that I wanted to set up OpenAI's API. The potential from there was huge. I decided I wanted to have some input sent in, and some output being returned. I reflected on making a code helper, that explains code, but there are already so much better resources on this. I realized that my competitive advantage would be something that ChatGPT couldn't do directly. You see, when you have almost no time, and you need to get some stuff done, you don't want to write a long message to ChatGPT explaining what you want it to do, what tasks you have to do, how much time you have, your preferences, and who you are. You want the AI to already know this. This is why I implemented both a personal schema, and a layout schema, so that the AI can help immediately.

### Features:
- **User Authentication**: Users can register, log in, and manage their profiles. Once logged in, they can update personal details, including age, gender, and life situation.
- **Plan Generation**: Users can create a personalized productivity plan based on their input (time available, tasks). OpenAI generates the plan, which is then stored and displayed.
- **History**: Users can view their previous plans in a list, with details about when they were created. This is mostly used as a fun feature for looking back at you past usage of the app.
- **Custom Schema**: Users can customize their productivity assistant’s layout by editing the schema that drives the generation of their plans. This will work as their personal preferences. Do the users prefer doing the hard tasks first, or save them for last. Are the users time efficient, or do they like to take their time. All of this is now custimizable by the users.

### Files Overview:
- **app/__init__.py**: Initializes the Flask application, including configurations for the database, login manager, and other global extensions.
- **app/routes.py**: Contains the main route for the app, the home page.
- **app/planner.py**: Contains the logic for generating and storing plans, interacting with OpenAI's API. Here we manage the schema and history for each user.
- **app/auth.py**: Managing the user registration, login, and logout functionality. Also handles user profile updates.
- **app/models.py**: Defines the database models, including `User`, `Schema`, `History`, and `Plan`. This file handles how data is stored and managed in the SQLite database.
- **app/forms.py**: Contains the Flask-WTF forms for user registration, login, plan input, and schema and profile management.
- **templates/**: Includes the HTML templates for each page in the app, using Bootstrap classes for styling.
  - **layout.html**: The base layout for the site, providing a consistent structure across pages. The navbar, flash message display, and shared layout structure.
  - **home.html**: The landing page for the app that provides links to key sections.
  - **make_plan.html**: The form where users input their available time and tasks to generate a plan.
  - **history.html**: Displays the list of plans previously generated by the user.
  - **profile.html**: The page for users to update their personal information.
  - **login.html**: The login page.
  - **register.html**: The user registration page.
  - **schema.html**: The page where users can edit their personal schema, which influences the layout of their productivity plans.
- **static/style.css**: Custom CSS to enhance the look and feel of the app. (Though Bootstrap is mainly used for styling).

### Technology Usage:
The project was designed to focus on simplicity and usability, providing the core functionality to generate and view productivity plans while ensuring an intuitive user experience. Some design choices include:

- **Use of Flask**: Flask was chosen due to its lightweight nature, flexibility, and ability to integrate easily with other libraries. It's also a framework I have worked with before (mainly in cs50), and I am comfortable with.
- **User Authentication**: Flask-Login was used for session management to ensure that only authenticated users can generate and access their personalized plans. I used their documentation to implement the different details (https://flask-login.readthedocs.io/en/latest/)
- **Form Handling**: Flask-WTF was used for form validation to ensure that all user input is secure and sanitized, especially when dealing with sensitive information like passwords. The documentation helped me with the implementation (https://flask-wtf.readthedocs.io/en/1.2.x/)
- **OpenAI Integration**: OpenAI's language model was integrated to generate productivity plans based on user input, offering an intelligent way to organize tasks. I used both the documentation and Q&A with their ChatGPT (https://platform.openai.com/docs/overview and https://chatgpt.com/)
- **Flask-SQLAlchemy**: For managing the SQLite database with python classes. This was quite challenging, and I used a lot of different guides on implementation and understanding. (https://flask-sqlalchemy.readthedocs.io/en/stable/ and https://www.geeksforgeeks.org/python/connect-flask-to-a-database-with-flask-sqlalchemy/ and https://www.digitalocean.com/community/tutorials/how-to-use-flask-sqlalchemy-to-interact-with-databases-in-a-flask-application)
- **Bootstrap**: A CSS framework for responsive and modern UI components. I used version 5.3, implemented through their documentation and tips from ChatGPT (https://getbootstrap.com/docs/5.3/getting-started/introduction/ and https://chatgpt.com/)
- **SQLite**: A lightweight, local, relational database to store user information, plans, and history. Learned mostly through the finance project in cs50, about databases in general.

### Installation and Setup: (see full documentation on https://docs.github.com/en)

1. Clone the repository:
    ```bash
    git clone <repository-url>
    cd <project-directory>
    ```

2. Create a virtual environment:
    ```bash
    python -m venv venv
    source venv/Scripts/activate
    ```

3. Install the required dependencies:
    ```bash
    pip install -r requirements.txt
    ```

4. Set up environment variables for the Flask app:
- Set your SECRET_KEY for the Flask application.
- Set OPENAI_API_KEY to enable OpenAI integration. (I of course removed my own)

5. Run the app:
    ```bash
    flask run
    ```

6. Visit the application in your browser at http://127.0.0.1:5000/.
